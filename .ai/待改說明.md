## 問題清單

### P0-1：Dashboard 篩選狀態污染側邊欄導航

現象：

- 使用者從 Dashboard 卡片切入待辦、案件、覆核等頁面後，頁面 `_filter_key` 被設成 Dashboard 篩選。
- 之後從側邊欄直接進該頁，仍沿用舊 `_filter_key`。
- 新增的案件、待辦、意見若不符合舊篩選，刷新後就從畫面消失，看起來像資料沒保存。

證據位置：

- `src/taxops/ui/main_window.py`
  - `navigate_to(page_id, filter_key="")` 只在 `filter_key` 有值時呼叫 `page.set_filter(filter_key)`。
  - `_on_nav_changed()` 只呼叫 `_activate_page(idx)`，沒有清除 filter。
- `src/taxops/ui/pages/tasks_page.py`
  - `set_filter()` 設定 `self._filter_key`。
  - `refresh_context()` 只 reload，不清空 `_filter_key`。
- `src/taxops/ui/pages/engagements_page.py`
  - 同樣保留 `_filter_key`。
- `src/taxops/ui/pages/review_notes_page.py`
  - 同樣保留 `_filter_key`。
- `src/taxops/ui/pages/clients_page.py`
  - 也有 `_filter_key`，需一併檢查 lease expiring filter 是否殘留。

最小修正：

1. 建立明確頁面協議：有 Dashboard filter 的頁面都加 `clear_filter()`。
2. `MainWindow._on_nav_changed()` 代表使用者從側邊欄直接進入，應呼叫目前頁面的 `clear_filter()` 再 `refresh_context()`。
3. `MainWindow.navigate_to(page_id, filter_key=...)` 代表 Dashboard 或程式導向：
   - 有 `filter_key`：先進頁，再套 `set_filter(filter_key)`。
   - 無 `filter_key`：不要保留前一次 Dashboard filter。
4. 加測試：Dashboard 切待辦 overdue 後，再用側邊欄進待辦，必須顯示全部待辦。

驗收：

- 從 Dashboard 點「逾期待辦」只看到逾期。
- 再點側邊欄「待辦事項」必須回到全部待辦。
- 新增一筆非逾期待辦後不會消失。
- 同樣覆蓋案件管理、覆核意見、客戶租約到期篩選。

### P0-2：各頁缺少「查看全部」一致入口

現象：

- 使用者期待從各頁本身查看所有資料，不應只能靠 Dashboard 切換。
- 待辦頁已有「全部案件」概念，但 filter 污染後失效。
- 案件頁目前只有客戶 combo，沒有「全部客戶」選項。
- 附件頁 combo 有「全部案件」文字，但 `_load_attachments()` 在 `_ALL` 時不載入任何附件。
- 索件管理頁目前強依賴 `load_engagement(engagement_id)`，側邊欄直接進入時沒有全域索件清單模式。

證據位置：

- `src/taxops/ui/pages/engagements_page.py`
  - `_client_combo` 只加入 clients，無 `_ALL_CLIENTS`。
  - `_refresh_engagements()` 在無 Dashboard filter 時只呼叫 `list_by_client(self._current_client_id)`。
- `src/taxops/ui/pages/attachments_page.py`
  - `_eng_combo.addItem("全部案件", _ALL)` 類似入口存在。
  - `_load_attachments()` 只有 `if eng_id and eng_id != _ALL` 才載入。
- `src/taxops/ui/pages/document_requests_page.py`
  - 只有 `load_engagement()` 設定 context。
  - 側邊欄直接進入不會顯示所有索件批次。

最小修正：

1. 案件頁新增「全部客戶」選項，底層用 `EngagementsService.list_all()`。
2. 附件頁 `_ALL` 時新增 repository/service `list_all()` 或逐案件彙整，不要顯示空表。
3. 索件管理頁做「全域模式」：
   - 側邊欄進入時顯示所有未刪除 document_requests。
   - 從案件進入時保留單案件 context。
   - 頁首清楚顯示目前是「全部索件批次」或「某案件索件批次」。
4. 每頁新增明確「清除篩選 / 查看全部」入口。

驗收：

- 每個主功能頁從側邊欄進入，都能看到該功能全部資料。
- Dashboard 篩選不會污染側邊欄進入的視圖。
- 新增資料後刷新仍在合理的目前視圖內可見。

### P1-2：索件管理文件項目只有單筆新增，沒有批量輸入、編輯、刪除

現象：

- 文件項目只能單獨輸入。
- 輸入錯誤後沒有編輯。
- 不需要的項目不能刪除。
- 沒有批量貼上，多項目索件很痛苦。

證據位置：

- `src/taxops/ui/dialogs/add_document_item_dialog.py`
  - 單一 `QLineEdit`，只呼叫 `DocumentRequestsService.add_item()`。
- `src/taxops/services/document_requests.py`
  - 有 `add_item()`、`set_item_status()`。
  - 沒有 `update_item_name()`。
  - 沒有 `delete_item()`。
  - 沒有 `add_items_bulk()`。
- `src/taxops/repositories/document_requests.py`
  - 需檢查是否已有 update/delete item；若沒有，補 repository 方法。

最小修正：

1. 新增 service/repository：
   - `add_items_bulk(request_id, raw_text)`：每行一項，trim，去空白，保留順序。
   - `update_item(item_id, item_name, notes=None)`。
   - `delete_item(item_id)`：建議軟刪或至少 audit；如果 schema 無 deleted_at，需新增 migration 或使用 status `not_applicable` 不等於刪除，需產品決策。
2. UI：
   - 「新增文件項目」改成多行 dialog，支援貼上一批。
   - 新增「編輯項目」「刪除項目」。
   - 刪除需二次確認。
3. 錯誤處理：
   - 空白行忽略但要顯示新增幾筆。
   - 全空要拒絕。
   - 超長項目要逐筆報錯，不可半成功假成功。

驗收：

- 貼上 5 行新增 5 個 items，順序一致。
- 空白行不新增。
- 全空拒絕且 DB 不寫入。
- 編輯項目名稱後 DB 更新、audit 有紀錄。
- 刪除項目後清單消失、request status 重新計算、audit 有紀錄。
